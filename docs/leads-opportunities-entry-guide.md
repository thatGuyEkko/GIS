## 一、概述

本模块已调整为**只读渲染**：页面直接读取两份 CSV 渲染，不提供在线新增 / 编辑入口，所有维护通过编辑 CSV 完成。

1. **data/leads.csv**：维护线索池数据（线索阶段状态固定为「待认领」，无状态列）。
2. **data/opportunities.csv**：维护在管商机与关闭样本，靠「状态」列判定归类。
3. 两份文件按各自 id 区分记录，线索 / 商机 / 关闭样本分别使用 **L- / O- / C-** 前缀；列表三栏（线索池 / 在管商机 / 关闭样本）由「文件来源 + 状态」自动生成。

## 二、页面取数逻辑

**三栏划分**

- **线索池**：来自 leads.csv 的全部记录，状态统一渲染为「待认领」。
- **在管商机**：来自 opportunities.csv、且状态不属于关闭状态的记录。
- **关闭样本**：来自 opportunities.csv、状态为 筛查未通过 / 初评未通过 / 已放弃 之一。

**指标卡与表格取数**

顶部 4 张指标卡（线索总数 / 待认领线索 / 在管商机 / 关闭样本）实时计算，不手填。下方表格列依次为：名称 / 国家 / 类型 / 状态 / 归属BG / 评分 / 负责人 / 附件 / 创建时间 / 更新时间。类型 / 归属BG / 来源 / 状态 均以带颜色标签展示。

## 三、CSV 字段填写

### 3.1 leads.csv（线索）

| --- | --- | --- | --- | --- |

| name | 线索名称 | 是 | 文本 | 标题，列表与表格主名称。为空不允许提交。 |
| country | 目标国家/城市 | 否 | 文本 | 如「泰国曼谷」「印尼」。 |
| type | 类型 | 否 | 枚举 6 选 1 | 见「四、类型」。 |
| bg | 归属BG | 否 | 枚举 9 选 1 | 见「四、归属BG」。留空即「未指定」。 |
| source | 来源 | 否 | 枚举 11 选 1 | 见「四、来源」。 |
| copy | 线索描述 | 否 | 长文本 | 一句话说明来源与价值，展示在副标题。 |
| attach | 附件 | 否 | 链接（可多个） | 填可访问 URL；多个用换行/分号/竖线分隔，渲染为可点击链接（新窗口打开）。 |
| created | 创建时间 | 否 | YYYY-MM-DD | 新增时自动填当天；历史数据可手动填。 |
| updated | 更新时间 | 否 | YYYY-MM-DD | 新增时自动填当天。 |

**当前表头**

```
id,name,country,type,bg,source,copy,attach,created,updated
```

### 3.2 opportunities.csv（商机 + 关闭样本）

| CSV 字段（英文） | 内部字段（中文映射） | 必填 | 类型 / 枚举 | 说明 |
| --- | --- | --- | --- | --- |
| id | 记录ID | 否 | 文本，前缀 O-/C- | 商机(O-)与关闭样本(C-)统一在此文件，手动维护需唯一。 |
| name | 商机名称 | 是 | 文本 | 必填，为空不允许提交。 |
| country | 目标国家/城市 | 否 | 文本 | 如「日本关西」。 |
| type | 类型 | 否 | 枚举 6 选 1 | 与线索共用同一套类型枚举，见「四、类型」。 |
| bg | 归属BG | 否 | 枚举 9 选 1 | 见「四、归属BG」。 |
| source | 来源 | 否 | 枚举 11 选 1 | 见「四、来源」。 |
| owner | 负责人 | 否 | 文本 | 如「张三」，填在表格「负责人」列。 |
| score | 商机评分 | 否 | 数字 0.0–5.0 | 步进 0.1；≥3.5 绿、≥2.5 黄、<2.5 红。 |
| copy | 商机描述 | 否 | 长文本 | 展示在卡片与表格副标题。 |
| attach | 附件 | 否 | 链接（可多个） | 填可访问 URL；多个用换行/分号/竖线分隔，渲染为可点击链接（新窗口打开）。 |
| status | 状态 | 是 | 枚举 9 选 1 | 见「四、状态」。关闭样本需填三个关闭值之一。 |
| created | 创建时间 | 否 | YYYY-MM-DD | 新增时自动填当天。 |
| updated | 更新时间 | 否 | YYYY-MM-DD | 新增时自动填当天。 |

**当前表头**

```
id,name,country,type,bg,source,owner,score,copy,attach,status,created,updated
```

> 商机 CSV 中的「类型」与线索「类型」是同一套枚举（投并购标的 / OTA平台 / 供应链整合 / 支付金融 / 软硬件销售 / 酒店管理），按类型枚举填写即可。

关闭样本（示例 C-001~C-003）与普通商机同处一份 opportunities.csv，仅「状态」列填三个关闭值之一，页面自动归入「关闭样本」栏，无需单独文件或类别标记。

### 3.3 附件填写规范（线索 / 商机共用）

「附件」列用于挂载与本条记录相关的可访问链接（方案文档、调研报告、合同扫描件等的分享地址），不是本地文件上传。

- **填什么**：一个或多个以 `http://` / `https://` / `ftp://` 开头的 URL。
- **多个链接**：用换行、英文分号 `;` 或竖线 `|` 分隔，页面逐个渲染为可点击链接，点击在新窗口打开。
- **留空**：不填则不显示附件区。
- **非链接文本**：若内容不是合法 URL（不以上述协议开头），按纯文本展示，不会变成链接，也不执行脚本（防 XSS）。
- **单条 URL 内可含逗号**（如查询参数 `?a=1,2`）；附件列用换行 / 分号 / 竖线分隔，不与 CSV 列分隔冲突。
- **示例**：`https://example.com/kansai-hotel-deal` 或 `https://a.com/x ; https://b.com/y`

## 五、ID 规则

| 类别 | 前缀 | 生成方式 |
| --- | --- | --- |
| 线索 | L- | 新增时自动 L- + 时间戳 |
| 商机 | O- | 新增时自动 O- + 时间戳 |
| 关闭样本 | C- | 手动在 CSV 中写为 C- 前缀的任意唯一值（如 C-20260101） |

## 六、CSV 格式要求

- **编码**：UTF-8（带 BOM 也可，页面解析时自动去除）。
- **分隔符**：英文逗号 `,`。
- **文本含逗号、换行或双引号时**，必须用英文双引号包裹；字段内双引号写成两个双引号 `""`。
- 不要有多余空行或表头拼写错误，否则该列会被忽略或错位。

## 七、校验规则

| 规则 | 说明 |
| --- | --- |
| 名称必填 | 线索名称 / 商机名称 建议必填；为空则该条记录名称为空。 |
| 商机评分范围 | 建议 0–5、步进 0.1；非数字会原样展示。 |
| 枚举约束 | 直接编辑 CSV 可填任意文本，页面原样展示（标签可能落入默认灰，建议严格按四枚举填）。 |
| 日期格式 | 不强制标准日期；非 YYYY-MM-DD 会原样展示，建议统一 YYYY-MM-DD。 |

## 八、本地预览与运行要求

页面通过 fetch 读取 `data/` 下的 CSV 渲染，**必须经 HTTP 访问**，双击 `file://` 打开会因浏览器安全策略读不到数据。

数据目录由页面脚本**按自身位置自动推算**（取「`assets/` 的同级目录 `data/`」），因此无论以本地根目录、GitHub Pages 子路径（`/GIS/`）还是子目录页面（`bp-details/`）访问，都能自动命中正确的 `data/`，**无需任何手工配置或写死前缀**。

**方式一：本地静态服务（推荐用于录入 / 改数后即时预览）**

在 GIS 根目录起任意静态服务即可，不依赖特定服务：

```bash
cd ~/Desktop/GIS
python3 -m http.server 8000      # 端口 8000，文档根 = ~/Desktop/GIS
```

浏览器打开（端口按上面命令）：
- 商机看板（主入口）：`http://127.0.0.1:8000/index.html`
- 线索 / 商机浏览：`http://127.0.0.1:8000/leads.html`
- BP 详情浏览：`http://127.0.0.1:8000/bp.html`（BP 数据维护见同级 `bp-module-database-design.md`）

> 只要保持「`assets/` 与 `data/` 同级」的目录结构，换端口 / 换静态服务都照常工作。

**方式二：线上已部署版本（GitHub Pages）**

站点已发布在 GitHub Pages 子路径，团队成员可直接访问，无需本地起服务：

- 主入口：`https://thatguyekko.github.io/GIS/index.html`
- 线索 / 商机：`https://thatguyekko.github.io/GIS/leads.html`
- BP 详情：`https://thatguyekko.github.io/GIS/bp.html`

⚠️ 无论本地还是线上，都**不要双击 `file://` 打开** HTML——浏览器会禁止本地文件 fetch，页面会显示「加载失败」（属预期，非代码问题）。

## 九、常见错误

1. 直接双击 HTML（file://）打不开数据：需启本地 HTTP 服务。
2. 在 leads.csv 误填「状态」列：线索状态固定「待认领」，该文件无状态列，误加会被忽略。
3. 枚举用旧写法（如 OTA平台业务 / 软件销售 / 净瓶监控 / 万达酒馆 / 酒店BG）：请用最新枚举，旧值落入默认灰。
4. 关闭样本误放进 leads.csv：应写在 opportunities.csv，状态填三个关闭值之一。
5. CSV 字段含逗号却没加引号：含逗文本必须用双引号包裹整段，否则拆列错位。

## 十、给录入人员的最短版本（5 条）

1. 两份文件分工：leads.csv 管线索，opportunities.csv 管商机 + 关闭样本。
2. 线索状态固定「待认领」无状态列；商机 / 关闭有「状态」列。
3. 关闭样本靠状态判定（筛查未通过 / 初评未通过 / 已放弃），写在 opportunities.csv，无单独文件。
4. 枚举用最新一套：类型 6 / 归属BG 9 / 来源 11 / 状态 9。
5. 必须走 HTTP 打开页面；数据完全来自 CSV，直接改 CSV 即改页面。
